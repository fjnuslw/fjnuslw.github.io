我觉得值得看的点

它有意思的地方不只是“做了图”，而是把 Agent 运行时里那些平时藏在日志、请求体和客户端协议里的东西拆开了。对正在做桌面 Agent、工具调用、上下文压缩或缓存优化的人来说，这类材料比抽象概念更容易建立直觉。

  • 请求与响应里到底有哪些结构化字段。
  • Prompt cache 如何影响多轮对话成本。
  • 工具调用为什么是模型与客户端之间的契约。
  • 动态加载工具如何兼顾上下文和缓存前缀。
  • 权限系统如何把自动化能力放进可控边界。
  • Skills、压缩、搜索工具这些机制如何服务长期任务。

留给自己的提醒

读这类系统拆解时，我最想带走的是“工程边界感”：模型能提出意图，客户端要负责执行、记录、审计和兜底。真正稳定的 Agent 不是把所有内容塞进 prompt，而是把状态、权限、工具和证据组织成模型能可靠使用的结构。
