Local Window Copilot

一个本地、轻量、低打扰的桌面陪伴伙伴。它不把“当前屏幕摘要”当成产品本体，而是把截图、VLM、观察记录和记忆放在后台；用户需要时，再通过可审计证据认真回答。

Current Direction

从“屏幕总结按钮”转向“在旁边的伙伴”

这个项目早期验证了窗口截图、MiniCPM-V 识图和本地问答链路，但继续围绕“摘要”推进会变得机械。现在的主线更明确：桌宠平时安静存在；观察线只生成证据；对话线默认不塞窗口摘要，模型需要时自己调用检索工具取证；调试线把所有观察、截图和工具轨迹展示给开发者。

低打扰

默认沉默，不连续汇报屏幕。主动提示必须短、少、确定有帮助。

按需取证

窗口观察不再固定注入 prompt；用户问题需要证据时，模型调用检索工具。

诚实边界

缺截图、缺观察、工具失败都明确暴露，不用泛泛回答伪装理解。

Architecture

三条线：观察、对话、调试

当前版本不追求复杂 planner，也不把一堆上下文硬塞给模型。系统被拆成三条责任清晰的链路：观察线保存可审计证据，对话线组织用户问题和工具调用，WebUI 调试线回答“到底看到了什么、查了什么、为什么这么答”。

观察线

只负责把目标窗口转成结构化证据，不负责和用户聊天。

目标窗口截图 + metadata
MiniCPM-V 结构化观察
SQLite 记录 + 截图文件

对话线

默认只有 system、profile、当前会话历史和用户问题。

基于证据流式回答

Mascot States

状态是存在感，不是大段解释

桌宠状态主要承担“我在”“我注意到了”“我正在认真看”的轻量反馈。图片保持原始比例展示，避免旧版卡片在窄容器里被挤压变形。

Idle 默认陪伴，轻微呼吸，不主动汇报。
Observing 注意到窗口变化，生成后台观察证据。
Analyzing 用户邀请时认真分析当前问题。
Privacy 隐私模式，暂停观察和截图链路。
Error 暴露真实失败点，避免伪装能力。

Implementation

当前已经落地的能力

它不是一个云端 SaaS，也不是自动操作电脑的 Agent。它的工程重点放在本地链路、观察证据、短期会话和可调试性上。

原生桌面悬浮窗

Windows 置顶透明窗口，支持拖动、状态表情、独立聊天窗和恢复观察。

结构化视觉观察

窗口截图、标题、进程、bounds、hash 一起进入观察记录，保留不确定区域。

确定性证据检索

候选观察和记忆用 FTS5 BM25 排名，不复用小 VLM 做不稳定 JSON rerank。

WebUI 高级调试

配置、观察存储、运行日志、profile、runtime JSON 和工具 trace 可视化。

Mainline Contract

对话不默认吃观察，观察不自动变记忆

观察线
  target window screenshot
  -> VLM structured observation
  -> window:latest_analysis
  -> window:summaries
  -> screenshot PNG

对话线
  user question + profile + session history
  -> model decides whether to call memory.search(query)
  -> evidence records with source / record_id
  -> final answer

调试线
  WebUI shows raw observation fields,
  screenshot thumbnails and tool traces.
唯一模型可见工具

memory.search(query) 负责检索窗口观察、短期记忆、profile 和历史对话证据。

不做固定大礼包上下文

不再每次把 latest、recent、memory 全塞进 prompt，避免“看起来有上下文，实际不可控”。

不做自动电脑操作

可以观察、陪伴、解释、建议和记录，但不自动点击、输入、提交或替用户执行高风险动作。

Roadmap

接下来打磨的是主链路质量

观察触发

点击观察后稳定捕获用户真正想看的窗口，并返回 record_id、title 和截图 hash。

存储完整性

latest/history 写入完整结构化字段，截图路径和原始 JSON 都可追踪。

工具 trace

对话过程展示模型查了什么、拿到哪些证据、最终回答使用了哪些来源。

产品化边界

启动检查、模型上下文校验、错误状态和 transient 状态恢复继续补齐。
